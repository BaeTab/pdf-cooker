# 역할(Role)
당신은 웹 기술의 한계를 뛰어넘는 시니어 프론트엔드 개발자입니다. 보안과 프라이버시를 최우선으로 생각하며, 서버 없이 클라이언트 사이드에서 작동하는 고성능 웹 애플리케이션을 구축합니다.

# 프로젝트 개요
React와 Vite를 사용하여 **'보안 PDF 공작소 (Secure PDF Workshop)'**를 개발합니다.
이 서비스의 핵심 철학은 **"No Server Upload"**입니다. 사용자가 업로드한 파일은 오직 브라우저 메모리 내에서만 처리되어야 하며, 외부로 전송되어서는 안 됩니다.

# 기술 스택 (Tech Stack)
- **Core:** React, Vite (JavaScript)
- **Style:** Tailwind CSS (신뢰감을 주는 Blue & Gray 테마, 깔끔한 Drag & Drop UI)
- **PDF Logic:** 1. `pdf-lib`: PDF 병합(Merge), 분할(Split) 등 문서 구조 조작용
  2. `pdfjs-dist`: PDF를 화면에 렌더링하거나 이미지로 변환(PDF to JPG)하는 용도
- **Utils:** `file-saver` (파일 다운로드), `jszip` (여러 이미지를 압축해서 다운로드), `react-dropzone` (파일 업로드 UX)

# 핵심 기능 요구사항 (Functional Requirements)

1. 네비게이션 (Tabs)
   - 상단에 3개의 메인 탭을 배치하세요: [PDF 합치기] / [PDF 나누기] / [PDF를 이미지로]

2. PDF 합치기 (Merge) 기능
   - **업로드:** 여러 개의 PDF 파일을 한 번에 드래그 앤 드롭으로 업로드.
   - **순서 변경:** 업로드된 파일 목록을 보여주고, 위/아래 버튼(또는 드래그)으로 합쳐질 순서를 변경 가능해야 함.
   - **실행:** '합치기' 버튼 클릭 시 `pdf-lib`를 사용하여 하나의 파일로 병합 후 다운로드.

3. PDF 나누기 (Split) 기능
   - **업로드:** 하나의 PDF 파일 업로드.
   - **페이지 추출:** - 옵션 A: "모든 페이지를 낱개로 분리"
     - 옵션 B: "특정 페이지만 추출" (예: 1, 3-5 입력 시 해당 페이지만 추출하여 새 PDF 생성)
   - **실행:** 결과물을 즉시 다운로드.

4. PDF를 이미지로 변환 (Convert to Image) 기능
   - **렌더링:** `pdfjs-dist`를 사용하여 PDF의 각 페이지를 HTML5 Canvas에 그림.
   - **변환:** Canvas의 내용을 JPG/PNG Blob으로 변환.
   - **압축 저장:** 변환된 이미지가 여러 장일 경우 `jszip`을 사용하여 `images.zip` 파일 하나로 압축해서 다운로드.

# UI/UX 디자인 가이드
- **Drag & Drop Zone:** 파일 업로드 영역은 점선 테두리와 아이콘을 사용하여 직관적으로 만드세요. 파일을 놓았을 때 색상이 변하는 인터랙션을 추가하세요.
- **Loading State:** PDF 처리 작업은 시간이 걸릴 수 있으므로, 처리 중일 때는 반드시 '스피너'나 '진행률 바'를 보여주어 사용자가 멈춘 것으로 오해하지 않게 하세요.
- **Privacy Badge:** "이 파일은 서버로 전송되지 않습니다"라는 문구와 자물쇠 아이콘을 잘 보이는 곳에 배치하여 안심시키세요.

# 기술적 주의사항 (중요)
- `pdfjs-dist`를 Vite 환경에서 사용할 때 Worker 설정이 까다로울 수 있습니다. CDN을 이용하거나 Vite 설정에 맞는 Worker 로드 방식을 적용하여 에러가 나지 않도록 신경 써주세요.
- 대용량 파일 처리를 고려하여 메모리 누수를 방지하는 코드를 작성하세요.
